# Hardware Radar Principles

**Version:** 1.0

> Every principle exists for one reason:
>
> **Help people buy with confidence.**

---

# HR-P001
## Truth Before Speed

Never publish information we are not reasonably confident is accurate.

Being first is less important than being trustworthy.

If data is stale, clearly communicate that it may be outdated.

Trust is earned through consistency.

---

# HR-P002
## Simplicity Wins

People visit Hardware Radar because they want answers.

Not complexity.

Every page should answer the user's primary question in under 30 seconds.

If a feature slows understanding, it should be reconsidered.

---

# HR-P003
## Transparency

Never hide how recommendations are made.

Always explain:

- Last verification
- Data source
- Affiliate relationships
- Recommendation methodology

Users deserve honesty.

---

# HR-P004
## Trust Before Revenue

Affiliate commissions never determine rankings.

The best recommendation always comes first.

Revenue follows trust.

Trust never follows revenue.

---

# HR-P005
## Data Is The Product

The website is simply the interface.

The true product is:

- accurate specifications
- verified pricing
- historical pricing
- compatibility
- recommendations
- buying intelligence

Every improvement should strengthen the quality of the underlying data.

---

# HR-P006
## Automation Exists To Improve Quality

Automation is never built simply to reduce work.

Automation exists to make Hardware Radar:

- faster
- more accurate
- more current
- more reliable

Manual verification always remains an option.

---

# HR-P007
## Build For Scale

Every system should be designed as though Hardware Radar will eventually contain tens of thousands of products.

Avoid shortcuts that limit future expansion.

Today's RAM database becomes tomorrow's hardware platform.

---

# HR-P008
## Trust Compounds

Trust grows through thousands of small improvements.

Correct one specification.

Verify one price.

Improve one explanation.

Every update should leave Hardware Radar more trustworthy than yesterday.

---

# HR-P009
## Decisions Over Clicks

Never optimize for clicks when we can optimize for better purchasing decisions.

Clicks create traffic.

Better decisions create loyal users.

Loyal users build enduring products.

---

# HR-P010
## Respect The User's Time

Every unnecessary click is friction.

Every unnecessary paragraph is noise.

Every unnecessary advertisement reduces trust.

Respecting the user's time is respecting the user.

---

# HR-P011
## Data Before Opinion

Whenever objective information exists, present it.

Opinions should complement data.

They should never replace it.

---

# HR-P012
## The Long Game

Hardware Radar is built for decades.

Every decision should strengthen the long-term reputation of the platform.

Never sacrifice future trust for short-term growth.

---

# Final Principle

Whenever uncertainty exists, ask one question.

> **Will this help someone buy with confidence?**

If the answer is yes—

Build it.

If the answer is no—

Don't.
