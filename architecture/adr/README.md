# Architecture Decision Records

Architecture Decision Records (ADRs) preserve the reasoning behind significant engineering decisions.

Code explains how a system works.

Architecture explains how systems interact.

ADRs explain why important decisions were made.

Whenever a decision is expected to influence the future architecture of Hardware Radar, an ADR should be created.

An ADR should be concise, easy to understand, and remain valuable years after it is written.

---

## ADR Format

Each ADR contains:

- Decision ID
- Status
- Context
- Decision
- Consequences
- Alternatives Considered
- Related Architecture Documents

The purpose is not to document every implementation detail.

The purpose is to preserve engineering intent.
